The HEBBS CLI (`hebbs-cli`) provides full access to the HEBBS engine from your terminal. It supports two modes of operation: one-shot commands for scripts and automation, and an interactive REPL for exploration and debugging.

## Two Modes

### One-Shot Commands

Run a single command and exit. Ideal for shell scripts, CI pipelines, and automation:

```bash
hebbs-cli remember --entity customer-42 --content "Prefers email communication"
hebbs-cli recall --entity customer-42 --query "contact preferences"
```

### Interactive REPL

Launch an interactive session with tab completion, command history, and dot-commands:

```bash
hebbs-cli repl
```

The REPL maintains connection state and provides a faster feedback loop for development and exploration.

## Key Capabilities

- **All 9 HEBBS operations** — remember, recall, revise, forget, prime, reflect, insights, subscribe, and set-policy are all available as CLI commands.
- **Multiple output formats** — human-readable tables (default), JSON for scripting, and raw format for piping.
- **Connection management** — configure server address, TLS, and timeouts via flags or environment variables.
- **Streaming support** — the `subscribe` command opens a live stream and prints memory pushes as they arrive.

## Authentication

The CLI requires an API key when the server has auth enabled (the default). Set it via environment variable or flag:

```bash
export HEBBS_API_KEY="hb_..."
hebbs-cli remember "hello world"

# Or per-command:
hebbs-cli --api-key "hb_..." remember "hello world"
```

The server prints a bootstrap API key on first start. See [Running the Server](/server/running/) for details.

## Configuration

The CLI reads configuration from (in priority order):

1. Command-line flags (`--address`, `--api-key`, `--tls`, `--timeout`)
2. Environment variables (`HEBBS_API_KEY`, `HEBBS_ADDRESS`, `HEBBS_TLS_CERT`, `HEBBS_TIMEOUT`)
3. Config file (`~/.config/hebbs/config.toml`)
