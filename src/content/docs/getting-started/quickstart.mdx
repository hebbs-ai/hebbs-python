<TabItem label="Vault (Local)">

## 1. Install

```bash
brew install hebbs-ai/tap/hebbs
```

Or via curl:

```bash
curl -sSf https://hebbs.ai/install | sh
export PATH="$HOME/.hebbs/bin:$PATH"
```

## 2. Initialize a Vault

```bash
cd ~/projects/my-notes    # any directory with markdown files
hebbs init --provider openai --model gpt-4o-mini --api-key-env OPENAI_API_KEY
```

This creates a `.hebbs/` directory, validates LLM connectivity, and configures the embedding model. HEBBS uses the LLM for proposition extraction, entity detection, and contradiction resolution during indexing.

**Embedding model**: By default, HEBBS uses OpenAI's `text-embedding-3-small` when `--provider openai` is set. You can override this with `hebbs config set embedding.model <model>`. For local-only embeddings (no API calls for embedding), use `hebbs config set embedding.model embeddinggemma-300m`.

Other providers:

```bash
# Anthropic
hebbs init --provider anthropic --model claude-haiku-4-5-20251001 --api-key-env ANTHROPIC_API_KEY

# Google Gemini
hebbs init --provider gemini --model gemini-2.0-flash --api-key-env GEMINI_API_KEY

# Local (Ollama, no API key needed)
hebbs init --provider ollama --model qwen3:4b
```

## 3. Index Your Files

```bash
hebbs index
```

HEBBS scans all `.md` files, extracts propositions and entities via the LLM, embeds everything, and stores triple-layer memories (document, propositions, entity/relation edges). Extraction runs in parallel (10 concurrent requests) for speed.

## 4. Recall

```bash
hebbs recall "meeting notes from last week"
```

## 5. Watch for Changes

```bash
hebbs watch
```

Now any file you create or edit is automatically re-indexed.

## 6. Open the Memory Palace

```bash
hebbs panel
```

A browser opens showing your memory graph with clusters, edges, and live updates.

## 1. Install

```bash
curl -sSf https://hebbs.ai/install | sh
export PATH="$HOME/.hebbs/bin:$PATH"
```

Verify:

```bash
hebbs-server --version
hebbs-cli --version
```

## 2. Start the Server

```bash
hebbs-server
```

In a new terminal, verify:

```bash
curl http://localhost:6381/v1/health/ready
```

## 3. Set Your API Key

On first start, HEBBS prints a bootstrap API key:

```

╚══════════════════════════════════════════════════════════════════╝
```

**Save this key.** Export it:

```bash
export HEBBS_API_KEY="hb_..."
```

:::tip
For local development, disable auth with `HEBBS_AUTH_ENABLED=false hebbs-server`.
:::

## 4. Store a Memory

```bash
hebbs-cli remember "Met with the Acme Corp team about Q2 roadmap."
```

## 5. Recall It

```bash
hebbs-cli recall "Acme meeting"
```

### Understanding the Score

The `Score` column is a **composite score**, not raw semantic similarity. HEBBS blends four signals:

| Signal | Default Weight | What it captures |
|--------|---------------|-----------------|
| **Relevance** | 0.5 | Cosine similarity between query and memory embeddings |
| **Recency** | 0.2 | How recently the memory was created |
| **Importance** | 0.2 | The memory's importance value |
| **Reinforcement** | 0.1 | How often the memory has been recalled before |

See [Recall Strategies](/concepts/recall-strategies/) for details on composite scoring.

## 6. Try the Python SDK

```bash
pip install hebbs
```

```python
import asyncio, os
from hebbs import HebbsClient

async def main():
    async with HebbsClient("localhost:6380", api_key=os.environ["HEBBS_API_KEY"]) as client:
        mem = await client.remember(content="My first memory from Python!", importance=0.8)
        print(f"Stored: {mem.id}")
        results = await client.recall("first memory", strategies=["similarity"])
        for r in results.results:
            print(f"  [{r.score:.2f}] {r.memory.content}")

asyncio.run(main())
```

## 7. Try the TypeScript SDK

```bash
npm install @hebbs/sdk
```

```typescript
import { HebbsClient } from '@hebbs/sdk';

const client = new HebbsClient('localhost:6380', { apiKey: process.env.HEBBS_API_KEY });
await client.connect();

const mem = await client.remember({ content: 'My first memory from TypeScript!', importance: 0.8 });
console.log(`Stored: ${mem.id.toString('hex')}`);

const { results } = await client.recall({ cue: 'first memory', strategies: ['similarity'] });
for (const r of results) console.log(`  [${r.score.toFixed(2)}] ${r.memory.content}`);

await client.close();
```
