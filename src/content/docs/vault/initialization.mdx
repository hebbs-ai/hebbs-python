## Creating a Vault

Initialize a vault in any directory. LLM configuration is required for new vaults:

```bash
cd ~/projects/my-notes
hebbs init --provider openai --model gpt-4o-mini --api-key-env OPENAI_API_KEY
```

Or specify a path:

```bash
hebbs init ~/projects/my-notes --provider anthropic --model claude-haiku-4-5-20251001 --api-key-env ANTHROPIC_API_KEY
```

The `init` command validates LLM connectivity before completing. If the provider is unreachable or the API key is invalid, initialization fails with a descriptive error.

### Embedding Model

When you set `--provider openai`, HEBBS defaults to OpenAI's `text-embedding-3-small` for embeddings. The embedding model and LLM are configured independently:

```bash
# Check current embedding config
hebbs config get embedding.model

# Switch to a different embedding model
hebbs config set embedding.model text-embedding-3-large

# Use local embeddings instead (no API calls, ~600MB model download)
hebbs config set embedding.model embeddinggemma-300m
hebbs config set embedding.dimensions 768
```

| Provider | Default Embedding Model | Dimensions |
|---|---|---|
| OpenAI | `text-embedding-3-small` | 1536 |
| Anthropic | `embeddinggemma-300m` (local) | 768 |
| Gemini | `embeddinggemma-300m` (local) | 768 |
| Ollama | `embeddinggemma-300m` (local) | 768 |

Anthropic and Gemini do not offer embedding APIs that HEBBS integrates with, so they default to the local ONNX model. OpenAI's embedding API is used automatically when OpenAI is the configured provider.

This creates a `.hebbs/` directory containing:

```
.hebbs/
  config.toml          # Vault configuration
  manifest.json        # File hash tracking (detects changes)
  index/
    db/                # RocksDB database (memories, edges, metadata)
    models/            # Only present when using local embeddings
      embeddinggemma-300m/
        config.json
        tokenizer.json
```

### Force Reinitialize

If `.hebbs/` already exists, `init` exits with an error. Use `--force` to reinitialize:

```bash
hebbs init --force
```

This resets `config.toml` to defaults and clears the manifest. The RocksDB database is preserved; use `hebbs rebuild` to start fresh.

## Global Brain

HEBBS supports a **global vault** at `~/.hebbs/` for memories that span all projects:

```bash
hebbs init --global
```

The global brain is useful for personal knowledge that is not project-specific. Use `--global` on any command to target it:

```bash
hebbs remember --global "Always use UTC for timestamps in APIs"
hebbs recall --global "timestamp conventions"
```

## Vault Discovery

When you run `hebbs recall` or any memory command without `--vault` or `--global`, HEBBS walks up the directory tree from the current working directory looking for a `.hebbs/` directory. This is the **project vault**.

The search order is:

1. `--vault /explicit/path` (if provided)
2. `--global` targets `~/.hebbs/`
3. Walk up from `$PWD` looking for `.hebbs/`
4. Fall back to `~/.hebbs/` if no project vault is found

## Multi-Vault Queries

Use `--all` on `recall` and `prime` to search both the project vault and the global brain, merging results by score:

```bash
hebbs recall --all "how to handle auth tokens"
```

This queries both vaults, deduplicates, and returns the top-k results sorted by composite score.

## After Initialization

Once initialized, index your files:

```bash
hebbs index
```

Then start recalling:

```bash
hebbs recall "your query here"
```

## Portable Cognition

`.hebbs/` is a self-contained cognition layer. Build the index once, then copy or share the directory across machines, agents, or team members. Everyone gets the same memory instantly.

Your markdown files are the source of truth. The `.hebbs/` directory is entirely derived and rebuildable. Delete it anytime and run `hebbs init && hebbs index` to reconstruct it from your files.

To control what gets indexed, create a `.hebbsignore` file at the vault root (same syntax as `.gitignore`). Your private files stay private; your agents only see what you allow. See [File Sync](/vault/file-sync/) for details on ignore patterns.

## .gitignore

Add `.hebbs/` to your `.gitignore`. The directory contains binary data (RocksDB, ONNX model weights) that should not be committed:

```
# HEBBS vault runtime data
.hebbs/
```
